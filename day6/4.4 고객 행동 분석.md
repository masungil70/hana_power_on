# 1-3. 고객 행동 분석

핵심은 “고객 단위로 데이터를 집계 → 행동 패턴 특징(feature) 생성 → 시각화” 흐름입니다.

---

# 1. 분석 목표

이 예제에서 하는 고객 행동 분석은 다음을 포함합니다.

* 고객별 총 소비 금액
* 고객별 거래 횟수
* 평균 거래 금액
* 소비 카테고리 분포
* 시간대별 소비 패턴
* 사기(fraud) 발생 고객 특징 비교

---

# 2. 전체 분석 코드 (Pandas + Matplotlib)

파일명 : customer_behavior_analysis.py

```python
import pandas as pd
import matplotlib.pyplot as plt

# =========================
# 1. 데이터 로드
# =========================
df = pd.read_csv("data/credit_card_transactions.csv")

# 불필요 컬럼 제거
df = df.drop(columns=["Unnamed: 0"])

# datetime 변환
df["trans_date_trans_time"] = pd.to_datetime(df["trans_date_trans_time"])

# 시간 파생 변수 생성
df["hour"] = df["trans_date_trans_time"].dt.hour
df["day"] = df["trans_date_trans_time"].dt.day_name()

# =========================
# 2. 고객 단위 행동 분석
# =========================

customer_group = df.groupby("cc_num")
```

이 코드는 DataFrame을 `cc_num`(카드 번호) 기준으로 그룹화(Grouping)합니다.

중요한 점은:

> 이 시점에서는 아직 계산이 수행되지 않습니다.

즉,

* sum
* mean
* count

같은 실제 연산은 아직 하지 않았고,

단지:

> "같은 카드 번호끼리 묶어둘 준비"

만 한 상태입니다.

## 6. 중요한 특징

### (1) 원본 DataFrame은 유지됨

```python

```

는 변하지 않습니다.

groupby는:

* 복사본 생성 X
* 그룹 정보만 관리

합니다.

---

### (2) Lazy Evaluation 느낌

```python

```

만으로는 실제 계산을 거의 하지 않습니다.

아래처럼 해야 계산 수행:

```python
customer_group.sum()
customer_group.mean()
customer_group.count()
```

## **그룹별 고객 행동을 요약 분석**

Pandas에서 **그룹별 고객 행동을 요약 분석**하는 전형적인 패턴입니다. 핵심은 `groupby`로 묶인 데이터에 대해 `agg()`로 여러 통계를 한 번에 계산하는 것입니다.

아래에서 구조를 하나씩 자세히 설명하겠습니다.

---

## 1. 구조 이해

```python
customer_behavior = customer_group.agg(
    total_spend=("amt", "sum"),
    avg_spend=("amt", "mean"),
    transaction_count=("amt", "count"),
    fraud_count=("is_fraud", "sum"),
)
```

👉 의미:
`customer_group`으로 묶인 고객별 데이터에 대해

* 총 소비
* 평균 소비
* 거래 횟수
* 사기 거래 횟수

를 한 번에 계산하는 코드입니다.

---

## 2. customer_group이 무엇인가?

보통 이런 형태입니다:

```python
customer_group = df.groupby("cc_num")
```

예:

* cc_num (카드 번호)
* 같은 고객끼리 그룹화

즉,
👉 “고객 단위로 데이터를 묶어라”

---

## 3. agg() 함수 의미

```python
.agg(...)
```

👉 여러 집계(aggregation)를 한 번에 수행하는 함수

기본 형태:

```python

```

## 4. 각 컬럼 상세 설명

---

### ① total_spend

```python
total_spend=("amt", "sum")
```

* amt 컬럼(거래 금액)의 합계
* 고객이 지금까지 사용한 총 금액

👉 의미:

> 고객별 총 소비 금액

예:

| 고객 | 거래금액 |
| -- | ---- |
| A  | 100  |
| A  | 200  |

→ total_spend = 300

---

### ② avg_spend

```python
avg_spend=("amt", "mean")
```

* 평균 거래 금액

👉 의미:

> 한 번 결제할 때 평균 얼마 쓰는가?

예:
| A | 100 |
| A | 200 |

→ 평균 = 150

---

### ③ transaction_count

```python
transaction_count=("amt", "count")
```

* amt 값의 개수
* 즉 거래 건수

👉 의미:

> 고객이 총 몇 번 결제했는가?

⚠️ 중요한 점:

* sum이 아니라 count
* null이 아닌 값 개수

---

### ④ fraud_count

```python
fraud_count=("is_fraud", "sum")
```

* is_fraud가 1인 값들의 합

보통 is_fraud 구조:

| is_fraud | 의미 |
| -------- | -- |
| 0        | 정상 |
| 1        | 사기 |

👉 sum을 하면:

* 1들의 개수 = 사기 거래 횟수

예:

| is_fraud |
| -------- |
| 0        |
| 1        |
| 1        |

→ fraud_count = 2

# 3. 결과 해석 (핵심 인사이트)

## 1) 고객 소비 패턴

* `total_spend` → 고객의 전체 소비 규모
* `transaction_count` → 활동성 (active user)
* `avg_spend` → 소비 성향 (소액 vs 고액 소비자)

👉 보통

* 고액 소비자 = 적은 거래 + 높은 평균 금액
* 일반 소비자 = 많은 거래 + 낮은 평균 금액

---

## 2) 이상 거래 탐지 포인트

```text
fraud_rate = fraud_count / transaction_count
```

* 특정 고객에서 fraud_rate가 높으면 위험 계정
* 고객 단위 risk scoring 가능

---

## 3) 시간대 분석 (중요)

* 새벽 시간대(0~4시)
  → 평균 거래 금액이 낮아야 정상
* 특정 시간대 급증
  → 자동화된 fraud 가능성

---

## 4) fraud vs normal 비교

* fraud 거래는 보통:

  * 평균 금액이 높거나
  * 특정 시간대에 집중됨

👉 이 차이가 ML feature 핵심

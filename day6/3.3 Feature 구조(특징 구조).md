# 1. Feature 구조란 무엇인가?

금융 데이터에서 **Feature 구조(특징 구조)**는 단순히 컬럼을 나열하는 개념이 아니라,
“거래 데이터를 모델이 이해할 수 있는 의미 단위로 분해한 결과”입니다.

즉, 원본 금융 데이터 → 머신러닝이 이해하는 숫자 벡터로 바꾸는 과정의 핵심이 Feature입니다.

금융 데이터에서 Feature는 이렇게 정의할 수 있습니다:

> “거래 1건을 설명하는 행동·환경·맥락 정보의 최소 단위”

예를 들어 원본 데이터:

```
amt=300000, category=electronics, time=03:12, city=Seoul
```

이걸 그대로 모델에 넣지 않고 다음처럼 바꿉니다:

```
amt, hour, category_encoded, distance, city_pop
```

👉 이 각각이 “Feature”입니다.

---

# 2. 금융 Feature 구조의 5가지 핵심 축

Feature를 기준으로 보면 금융 데이터는 크게 5가지 관점으로 구성됩니다:

---

# 2-1. 금액 Feature (amt)

## 의미

```text
amt = 거래 금액
```

하지만 단순 숫자가 아닙니다.

---

## 금융적 해석

* 소비 규모
* 이상 거래 여부 판단 기준
* 고객 경제 수준 반영

---

## 중요한 점 (핵심)

### ❌ 잘못된 해석

* 300,000원 = 큰 금액

### ✅ 올바른 해석

* “이 사람의 평균 대비 몇 배인가?”

---

## 실제 Feature 변환

```text
amt_ratio = amt / user_avg_amt
```

예:

| 고객 | 평균   | 현재    |
| ---- | ------ | ------- |
|   A  | 20,000 | 25,000  |
|   B  | 20,000 | 300,000 |

👉 B는 15배 이상 → 이상 가능성 ↑

---

# 2-2. 시간 Feature (hour)

## 의미

```text
hour = 거래 발생 시간 (0~23)
```

---

## 금융에서 중요한 이유

금융 거래는 “사람의 생활 패턴”과 연결됩니다.

### 정상 패턴

* 09:00 ~ 22:00 소비

### 위험 패턴

* 02:00 ~ 05:00 결제

---

## Feature 확장

단순 hour가 아니라 이렇게 변환됩니다:

```text
is_night (0/1)
is_weekend
hour_sin / hour_cos (주기성)
```

---

## 핵심 개념

👉 시간은 “숫자”가 아니라 “주기적 행동 패턴”

---

# 2-3. category Feature (카테고리)

## 의미

```text
category = 소비 유형 (식비, 쇼핑, 여행 등)
```

---

## 금융적 의미

카테고리는 “소비 목적”입니다.

| category    | 의미        |
| ----------- | ----------  |
| food        | 생활 소비   |
| travel      | 이동/여행   |
| electronics | 고위험 소비 |
| gambling    | 고위험      |

---

## 머신러닝 변환

문자 → 숫자로 변환해야 합니다.

### 방법 1: One-Hot Encoding

```
food = [1,0,0]
travel = [0,1,0]
```

### 방법 2: Target Encoding (고급)

```
category_fraud_rate
```

---

## 핵심

👉 category는 “돈이 어디로 흐르는가”를 나타내는 행동 지표

---

# 2-4. distance Feature (거리)

## 의미

```text
distance = 이전 거래 위치와 현재 거래 위치 거리
```

---

## 왜 중요한가?

사람은 물리적으로 이동해야 거래할 수 있습니다.

---

## 이상 탐지 핵심 규칙

### 정상

```
Seoul → Seoul (5km)
```

### 이상

```
Seoul → New York (10,000km, 10분 후)
```

---

## Feature 확장

```text
speed = distance / time_diff
impossible_move_flag
```

---

## 핵심

👉 distance는 “물리적 현실성” 검증 Feature

---

# 2-5. city_pop Feature (도시 인구)

## 의미

```text
city_pop = 해당 도시 인구 수
```

---

## 금융에서 왜 중요?

도시 규모는 “거래 위험도”와 연결됩니다.

---

## 해석

| 도시       | 의미                |
| ---------- | ------------------- |
| small town | 이상 거래 탐지 쉬움 |
| big city   | 정상 거래 많음      |

---

## 활용 방식

```text
risk_weight = 1 / log(city_pop)
```

또는

```
fraud_probability_adjustment
```

---

## 핵심

👉 city_pop은 “환경 기반 위험도 보정값”

---

# 3. Feature 구조 전체 그림

금융 Feature는 단순히 나열된 값이 아니라 이렇게 역할이 나뉩니다:

```
[금액]
   ↓
amt, amt_ratio

[시간]
   ↓
hour, is_night, cyclical encoding

[행동]
   ↓
category, category_risk

[공간]
   ↓
distance, location anomaly

[환경]
   ↓
city_pop
```

---

# 4. 머신러닝 관점에서 Feature의 의미

머신러닝 모델은 사실 이렇게 생각합니다:

> “이 거래가 정상인지 아닌지 판단할 수 있는 숫자 단서들”

---

## 예시 Feature 벡터

```
[300000, 3, 0.0, 12000km, 50000]
```

의미:

| Feature  | 의미        |
| -------- | ----------- |
| amt      | 금액        |
| hour     | 새벽        |
| category | 고위험      |
| distance | 비정상 이동 |
| city_pop | 환경 위험   |

---

# 5. 핵심 정리

금융 Feature 구조의 핵심은 3가지입니다:

---

## 1) 절대값이 아니라 “비율/변화”

* amt → amt_ratio
* 단순 숫자 X

---

## 2) 단일 값이 아니라 “맥락 정보”

* hour → 생활 패턴
* category → 행동 유형

---

## 3) 현실성 검증 요소 포함

* distance → 물리적 가능성
* city_pop → 환경 위험도

---

# 6. 한 문장으로 정리

> 금융 Feature 구조란 “거래 데이터를 금액·시간·행동·공간·환경이라는 다섯 축으로 분해하여, 머신러닝이 이상 여부를 판단할 수 있는 의미 있는 숫자 벡터로 변환하는 과정”입니다.

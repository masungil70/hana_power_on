# 1. Label 구조란 무엇인가?

금융 데이터 구조에서 **Label 구조(정답 구조)**는 머신러닝 모델의 성능을 결정하는 핵심 요소입니다. 특히 금융에서는 “예측 대상이 무엇인가?”를 정의하는 단계이기 때문에 단순한 컬럼 하나가 아니라 **비즈니스 의미 + 데이터 생성 방식 + 학습 구조 전체를 포함**합니다.

간단히 말하면:

> **Label = 모델이 맞혀야 하는 정답(타깃 변수)**

금융 데이터에서는 보통 아래처럼 정의됩니다:
    
| 입력 (X)                                | 출력 (Y = Label)     |
| --------------------------------------- | ----------------     |
| 거래 정보 (시간, 금액, 위치, 가맹점 등) | is_fraud (사기 여부) |

---

## 핵심 개념

* 입력: “이 거래가 어떤 특징을 가지고 있는가?”
* 출력: “이 거래는 정상인가 / 사기인가?”

👉 즉, 금융 ML 문제는 대부분 **이진 분류(Binary Classification)** 입니다.

---

# 2. 금융 Label 구조의 특징 (중요)

금융 Label은 일반 데이터와 다르게 다음 특징이 있습니다.

---

## 2-1. Label은 “결과”가 아니라 “판단된 기록”이다

예:

```text
is_fraud = 1 → 사기 거래로 판정됨
is_fraud = 0 → 정상 거래
```

하지만 중요한 점:

👉 이 값은 “모델이 만들어낸 값”이 아니라
👉 **과거 시스템 또는 전문가가 판정한 결과**

---

📌 즉 Label은 “**과거의 판단 데이터**”입니다.

---

## 2-2. Label은 매우 불균형(Imbalanced)하다

금융 데이터의 가장 중요한 특징:

| 정상 거래 | 사기 거래 |
| --------- | ----- |
| 99.8%     | 0.2%  |

---

👉 문제 발생:

* 모델이 전부 0(정상)만 예측해도 정확도 99% 나옴
* 하지만 실제로는 의미 없는 모델

---

📌 그래서 금융 ML에서는 모델의 성능을 평가하기 위해 사용되는 지표(Accuracy)보다:

* Precision (정확도)
* Recall (검출률)
* F1-score
* AUC-ROC

이 훨씬 중요합니다.

---

## 2-3. Label은 “지연 확정 데이터”일 수 있다

금융에서는 즉시 Label이 결정되지 않습니다.

예:

1. 거래 발생
2. 고객 신고
3. 조사 진행
4. 몇 일~몇 주 후 확정

👉 즉 Label은 “시간 지연 후 확정된 정답”

---

📌 이 때문에 발생하는 문제:

* 실시간 모델 vs 사후 Label mismatch
* 데이터 누수(Data Leakage) 위험

---

# 3. Label 생성 구조 (중요 개념)

금융 Label은 보통 3가지 방식으로 생성됩니다.

---

## 3-1. Rule 기반 Label

```text
if 해외 + 고액 + 짧은 시간 이동 → fraud = 1
```

👉 장점:

* 빠름
* 초기 시스템 구축 가능

👉 단점:

* 정밀도 낮음
* 패턴 변화 대응 어려움

---

## 3-2. 전문가 판정 Label

* 금융 보안팀
* 고객 신고
* 수사 결과

👉 가장 신뢰도 높은 Label

---

## 3-3. 시스템 자동 Label (ML 기반)

* 기존 모델이 1차 판단
* 이후 재검증

---

# 4. Label 구조를 잘못 이해하면 생기는 문제

---

## 4-1. 데이터 누수 (Data Leakage)

예:

```text
입력에 "fraud_flag_history" 포함
```

👉 문제:

* 미래 정보를 학습
* 모델 성능 과대평가

---

## 4-2. Label 정의 불일치

예:

* 어떤 팀: 10만원 이상이면 fraud
* 다른 팀: 100만원 이상

👉 결과:

* 모델 학습 데이터 혼란
* 성능 저하

---

## 4-3. Label 타이밍 문제

```text
거래 시점 vs 판정 시점 불일치
```

👉 실무에서는 가장 중요한 문제 중 하나

---

# 5. 금융 Label 구조의 실제 예시

## 원본 데이터

| cc_num | amt | category    | time  | is_fraud |
| ------ | --- | ----------- | ----- | -------- |
| 1111   | 20k | food        | 10:00 | 0        |
| 1111   | 2M  | electronics | 03:00 | 1        |

---

## 모델 학습 구조

### 입력 (X)

```text
amt
category
hour
user_avg_amt
distance
```

### 출력 (Y)

```text
is_fraud
```

---

## 모델이 배우는 것

👉 단순 암기가 아니라:

* “새벽 + 고액 + 비정상 카테고리 → 위험”
* “평소 대비 금액 급증 → 위험”

---

# 6. Label 구조를 확장하면 이렇게 된다 (실무)

금융에서는 단순 0/1이 아니라 확장됩니다.

---

## 6-1. 다중 Label 구조

```text
0 = 정상
1 = 의심
2 = 사기 확정
```

---

## 6-2. 확률 Label 구조

```text
fraud_probability = 0.87
```

👉 더 현실적인 구조 (현업 AI)

---

## 6-3. 시계열 Label 구조

```text
t 시점 거래 → t+7일 후 fraud 여부 확정
```

👉 Time-delayed Label

---

# 7. 핵심 요약

금융 데이터에서 Label 구조는 단순한 컬럼이 아니라:

> “과거 거래를 기반으로 사람이 또는 시스템이 판단한 위험 여부를 모델이 학습할 수 있도록 정의한 정답 체계”

---

# 8. 한 줄 핵심 정리

👉 금융 Label 구조 =
**“미래를 예측하기 위한 과거의 판단 기록이며, 대부분 매우 불균형하고 시간 지연 특성을 가진 정답 데이터”**

# 4. 고객 생애 기반 시간 분석 (Age × Time)

## 목적

* 연령대별 소비 시간 차이

---

### 📊 결과 예시

| Age Group | 주요 소비 시간 |
| --------- | -------------- |
| 20대      | 22~02시        |
| 30대      | 18~22시        |
| 40대      | 12~18시        |

---

```phthon
# -----------------------------
# 4. 나이 / 연령대 생성
# -----------------------------
df["dob"] = pd.to_datetime(df["dob"])

current_year = pd.to_datetime("today").year
df["age"] = current_year - df["dob"].dt.year

# 연령대 구간 생성
bins = [0, 20, 30, 40, 50, 60, 100]
labels = ["10대", "20대", "30대", "40대", "50대", "60대+"]
df["age_group"] = pd.cut(df["age"], bins=bins, labels=labels, right=False)

```

![alt text](image-1.png)

---

이 코드는 고객의 `age(나이)` 값을 기준으로 **연령대 그룹(age_group)** 을 생성하는 코드입니다.
즉, 숫자 형태의 나이를 `"20대"`, `"30대"` 같은 범주형 데이터로 변환합니다.

---

### 1. 구간(bins) 정의

```python
bins = [0, 20, 30, 40, 50, 60, 100]
```

이 부분은 나이를 어떤 기준으로 나눌지 정의합니다.

각 숫자는 **구간의 경계값(boundary)** 입니다.

구간은 아래처럼 생성됩니다.

| 구간 범위 | 의미      |
| --------  | ------    |
| 0 ~ 20    | 10대      |
| 20 ~ 30   | 20대      |
| 30 ~ 40   | 30대      |
| 40 ~ 50   | 40대      |
| 50 ~ 60   | 50대      |
| 60 ~ 100  | 60대 이상 |

즉:

```python
0 <= age < 20
20 <= age < 30
30 <= age < 40
...
```

형태로 나이를 분류합니다.

---

### 2. 각 구간에 붙일 이름(labels)

```python
labels = ["10대", "20대", "30대", "40대", "50대", "60대+"]
```

위에서 만든 각 구간에 대해 사람이 이해하기 쉬운 이름을 붙입니다.

구간 개수는:

```python
len(bins) - 1
```

개여야 합니다.

왜냐하면:

```python
[0,20]
[20,30]
[30,40]
...
```

처럼 구간은 경계값 사이에서 만들어지기 때문입니다.

현재:

* bins 개수 = 7개
* 실제 구간 개수 = 6개

따라서 labels도 6개입니다.

---

### 3. pd.cut() 으로 범주화

```python
df["age_group"] = pd.cut(
    df["age"],
    bins=bins,
    labels=labels,
    right=False
)
```

여기가 핵심입니다.

`pd.cut()` 함수는 연속형 숫자 데이터를 구간별로 잘라서 범주형 데이터로 변환합니다.

---

### 4. pd.cut() 구조

```python
pd.cut(
    데이터,
    bins=구간경계,
    labels=구간이름
)
```

### 4. right=False 의 의미

이 옵션이 매우 중요합니다.

```python
right=False
```

의 의미는:

```python
왼쪽 포함, 오른쪽 제외
```

입니다.

즉 구간이 아래처럼 됩니다.

```python
[0,20)
```

수학적으로:

```python
0 <= x < 20
```

를 의미합니다.

---

### 현재 구간 해석

| 구간 표현 | 실제 의미       |
| -------   | --------------- |
| [0,20)    | 0 이상 20 미만  |
| [20,30)   | 20 이상 30 미만 |
| [30,40)   | 30 이상 40 미만 |

즉:

* 20세 → 20대
* 30세 → 30대
* 40세 → 40대

로 들어갑니다.

---

### right=False 가 없으면?

기본값은:

```python
right=True
```

입니다.

그러면 구간이:

```python
(0,20]
```

가 됩니다.

즉:

```python
0 < x <= 20
```

의 의미입니다.

이 경우:

* 20세 → 10대
* 30세 → 20대

가 되어버립니다.

그래서 보통 연령대 분석에서는:

```python
right=False
```

를 많이 사용합니다.

### 범주형 데이터의 장점

범주형(category)은 분석에서 매우 중요합니다.

장점:

| 장점           | 설명              |
| -------------- | ----------------- |
| 메모리 절약    | 문자열보다 효율적 |
| 그룹 분석 빠름 | groupby 최적화    |
| 순서 관리 가능 | 연령대 순서 유지  |
| 시각화 편리    | 그래프 자동 정렬  |

---

### 금융 데이터 분석에서 왜 중요한가?

금융 데이터에서는 연령대별 소비 패턴 분석을 자주 합니다.

예시:

| 연령대  | 특징                |
| ------- | -----------         |
| 20대    | 온라인 소비 많음    |
| 30대    | 육아/생활 소비 증가 |
| 40대    | 고액 소비 증가      |
| 60대+   | 의료 소비 증가      |

즉:

```python
age_group = df.groupby("age_group")

```

이 코드는 DataFrame을 `age_group`(연령대별) 기준으로 그룹화(Grouping)합니다.

중요한 점은:

> 이 시점에서는 아직 계산이 수행되지 않습니다.

즉,

* sum
* mean
* count

같은 실제 연산은 아직 하지 않았고,

단지:

> "같은 연령대별끼리 묶어둘 준비"

만 한 상태입니다.

# 🔥 2. 가장 중요한 핵심 패턴

## ✅ (1) 20대: 밤 중심 소비 (21~24시 강함)

* 낮 시간: 거의 활동 없음 (어두운 색)
* 밤 시간: 급격히 증가 (밝아짐)

📌 해석

* 야간 소비 집중
* 온라인 쇼핑 / 배달 / 엔터 소비 중심

👉 특징

> “생활 리듬이 늦은 시간대로 밀려 있음”

---

## ✅ (2) 30대: 오후 ~ 저녁 중심 (15~22시)

* 낮보다 오후부터 증가
* 저녁에 최고점

📌 해석

* 직장인 패턴
* 퇴근 이후 소비 집중

👉 특징

> “가장 전형적인 안정형 소비 패턴”

---

## ✅ (3) 40대: 낮 + 저녁 혼합형

* 낮에도 일정 수준 소비 존재
* 저녁에도 유지

📌 해석

* 업무/가정 혼합 소비 구조
* 시간 분산형 소비

👉 특징

> “가장 균형적인 소비 구조”

---

## ✅ (4) 50대: 안정적 + 일정한 패턴

* 시간대 변화 크지 않음
* 전반적으로 일정

📌 해석

* 생활형 소비
* 계획적 소비 성향

👉 특징

> “변동성이 낮은 안정형 고객군”

---

## 🔴 (5) 60대+: 낮 시간 집중 (가장 강한 특징)

* 오전~오후 매우 높은 소비
* 밤에는 급감

📌 해석

* 활동 시간이 낮 중심
* 오프라인 소비 비중 높을 가능성

👉 특징

> “명확한 데이타임 소비 그룹”

### 개념

# 🚨 1. 거리 기반 시간 이상 탐지 개념

이 분석은 2가지 데이터를 결합합니다:

## 📌 사용 변수

* 고객 위치: `lat`, `long`
* 가맹점 위치: `merch_lat`, `merch_long`
* 시간: `trans_date_trans_time`

# 🧠 2. 핵심 아이디어

## 📌 Step 1: 이동 거리 계산

고객 → 가맹점 거리

## 📌 Step 2: 시간 차이 계산

이전 거래와 현재 거래 시간 차이

## 📌 Step 3: 속도 계산

Speed = Distance / Time

# ⚠️ 3. 이상 탐지 기준 (핵심)

| 시간  | 거리    | 해석                          |
| ----- | ------- | ----------------------------- |
| 5분   | 200km   | ❌ 물리적으로 불가능 → fraud |
| 30분  | 2km     | ⭕ 정상                      |
| 10분  | 500km   | ❌ 매우 강한 fraud           |

## 📌 1) 거리 계산 (Haversine)

```python
import numpy as np

def haversine(lat1, lon1, lat2, lon2):
    R = 6371  # 지구 반지름 (km)

    lat1, lon1, lat2, lon2 = map(np.radians, [lat1, lon1, lat2, lon2])

    dlat = lat2 - lat1
    dlon = lon2 - lon1

    a = np.sin(dlat/2)**2 + np.cos(lat1)*np.cos(lat2)*np.sin(dlon/2)**2
    c = 2 * np.arcsin(np.sqrt(a))

    return R * c
```

---

## 📌 2) 거리 계산 적용

```python
df["distance_km"] = haversine(
    df["lat"],
    df["long"],
    df["merch_lat"],
    df["merch_long"]
)
```

---

## 📌 3) 시간 차이 계산

```python
df = df.sort_values(by=["cc_num", "trans_date_trans_time"])

df["time_diff"] = df.groupby("cc_num")["trans_date_trans_time"].diff().dt.total_seconds() / 60
```

---

## 📌 4) 속도 계산

```python
df["speed_kmh"] = df["distance_km"] / (df["time_diff"] / 60)
```

---

## 📌 5) 이상 탐지 조건

```python
df["velocity_fraud"] = (
    (df["distance_km"] > 100) & (df["time_diff"] < 10)
)
```

금융 데이터에서 **시간(Time) 구조**는 단순히 “언제 발생했는가”를 넘어서,
**사람의 행동 패턴·이상 탐지·예측 모델의 핵심 기준축**이 됩니다.

특히 금융에서는 “시간 = 행동의 맥락”이기 때문에 가장 중요한 축 중 하나입니다.

아래에서 단계적으로 깊게 설명드리겠습니다.

---

# 1. 시간 데이터의 본질: “순서 + 맥락 + 패턴”

금융 데이터에서 시간은 단순 날짜가 아니라 다음 3가지를 포함합니다.

### ✔ ① 순서 (Order)

* 어떤 거래가 먼저 발생했는가?
* 이전 거래와 다음 거래 관계

📌 예:

* 10:00 → 10:02 → 10:03 연속 결제
  👉 자동결제/봇/사기 가능성

---

### ✔ ② 간격 (Interval)

* 거래 간 시간 차이

📌 예:

* 1시간 동안 20번 결제
  👉 비정상 행동 가능성

---

### ✔ ③ 맥락 (Context)

* “언제 했느냐”에 따라 의미가 완전히 달라짐

📌 예:

* 낮 2시 10만원 결제 → 정상
* 새벽 3시 10만원 결제 → 의심

---

# 2. 금융 데이터의 시간 컬럼 구조 해석

대표적으로 이런 컬럼이 있습니다:

```text
trans_date_trans_time
```

이 하나의 값 안에는 사실 여러 정보가 들어있습니다:

```
2026-05-11 03:12:45
```

👉 이걸 쪼개면:

| 구성     | 의미 |
| ------ | -- |
| year   | 연도 |
| month  | 월  |
| day    | 일  |
| hour   | 시간 |
| minute | 분  |
| second | 초  |

---

# 3. 파생되는 핵심 시간 Feature

금융 분석에서는 이 값을 그대로 쓰지 않고 **쪼개서 새로운 변수(feature)를 만듭니다.**

---

## 3-1. Hour (시간대)

```python
hour = trans_date_trans_time.hour
```

### 의미:

* 하루 24시간 중 언제 거래했는가

### 패턴:

| 시간    | 의미               |
| ----- | ---------------- |
| 09~18 | 정상 업무 시간         |
| 22~06 | 위험 시간 (fraud 많음) |

📌 예:

* 새벽 2시 해외 결제 → 위험 점수 증가

---

## 3-2. Day (일 단위 패턴)

```python
day = trans_date_trans_time.day
```

### 의미:

* 월 중 언제 소비가 집중되는가

### 패턴:

* 월 초: 급여 직후 소비 증가
* 월 말: 소비 감소 또는 카드 한도 초과 시도

---

## 3-3. Weekday (요일)

```python
weekday = trans_date_trans_time.weekday()
```

### 의미:

* 요일별 소비 패턴

### 일반 패턴:

| 요일  | 특징       |
| --- | -------- |
| 월~금 | 생활/업무 소비 |
| 토~일 | 여행/여가 소비 |

📌 이상 패턴:

* 평일 새벽 + 고액 여행 결제 → 의심

---

## 3-4. Month / Season (계절성)

```text
month = 1~12
```

### 의미:

* 계절/이벤트 기반 소비 패턴

### 예:

* 12월: 소비 증가 (연말)
* 2월: 감소
* 명절: 특정 카테고리 증가

---

# 4. 금융 데이터에서 시간의 핵심 개념: “비정상성”

금융 데이터는 **시간에 따라 패턴이 변하는 데이터(non-stationary)** 입니다.

즉:

👉 “항상 같은 패턴이 아니다”

---

## 예시:

### 평소 패턴

* 하루 2~3번 결제
* 평균 2만원

### 갑자기

* 5분 동안 10번 결제
* 200만원 사용

👉 시간 패턴이 깨짐 → Fraud 가능성 ↑

---

# 5. 핵심: “시간은 단독이 아니라 관계로 분석한다”

시간은 항상 다른 변수와 결합됩니다:

---

## 5-1. 시간 + 금액

```text
새벽 + 고액 결제
```

👉 가장 강력한 fraud 신호 중 하나

---

## 5-2. 시간 + 위치

```text
10분 전에 서울 → 현재 미국 결제
```

👉 물리적으로 불가능 → 사기 확률 ↑

---

## 5-3. 시간 + 빈도

```text
1분에 5번 결제
```

👉 자동화 공격 가능성

---

# 6. 고급 시간 Feature Engineering

실무에서는 단순 hour보다 훨씬 복잡한 feature를 만듭니다.

---

## 6-1. Time Difference (시간 간격)

```python
time_diff = current_time - previous_time
```

📌 의미:

* 거래 간 간격 분석

| 값     | 의미    |
| ----- | ----- |
| 0~10초 | 봇 가능성 |
| 1~10분 | 의심    |
| 몇 시간  | 정상 가능 |

---

## 6-2. Rolling Window (이동 통계)

```python
last_1h_transaction_count
last_24h_spend
```

👉 “최근 N시간 동안 얼마나 썼는가”

---

## 6-3. Burst Detection (폭주 탐지)

* 갑자기 거래량 증가

📌 예:

* 평소 2건 → 갑자기 20건

---

# 7. 시간 구조를 모델링에 쓰는 방식

## 7-1. 단순 모델 입력

```text
hour, weekday, month
```

👉 LightGBM / XGBoost

---

## 7-2. 시계열 모델

```text

```

👉 시간 순서 자체를 학습

---

## 7-3. 그래프 기반 모델

```text
user → merchant → time
```

👉 금융 네트워크 분석

# 8. 핵심 요약

금융 데이터에서 시간 구조는 단순한 날짜가 아니라:

> “사용자의 행동 패턴, 정상/비정상 판단, 미래 예측을 결정하는 핵심 축”

---

## 핵심 3가지

### ✔ 1. 시간 = 순서 정보

* 어떤 거래가 먼저 발생했는가

### ✔ 2. 시간 = 행동 패턴

* 언제 소비하는 사람인가

### ✔ 3. 시간 = 이상 탐지 기준

* “언제 발생했는가”가 정상/비정상을 결정

---

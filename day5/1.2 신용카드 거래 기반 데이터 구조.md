# 1 금융 거래 데이터 필드 구조

| 필드명                | 의미                      | 데이터 역할      | 분석/모델링 활용         |
| --------------------- | ------------------------- | ---------------- | -----------------------  |
| Unnamed: 0            | CSV 저장 시 생성된 인덱스 | 불필요한 식별자  | 제거 대상 (사용 X)       |
| trans_date_trans_time | 거래 발생 날짜/시간       | 시간 정보        | 시간대, 요일, 패턴 분석  |
| cc_num                | 신용카드 번호 (고객 ID)   | 고객 식별자      | 고객 단위 행동 분석      |
| merchant              | 가맹점 이름               | 거래 발생 장소   | 위험 가맹점 분석         |
| category              | 거래 유형                 | 소비 분류        | 소비 패턴 분석           |
| amt                   | 거래 금액                 | 핵심 수치 데이터 | 이상 거래 탐지 핵심 변수 |

| last                  | 고객 이름 (성)            | 고객 정보        | 보조 식별 (거의 미사용)  |
| gender                | 성별                      | 고객 속성        | 고객 그룹 분석           |
| street                | 주소 (도로명)             | 위치 정보        | 지역 분석                |
| city                  | 도시                      | 위치 정보        | 지역 패턴 분석           |
| state                 | 주/도                     | 위치 정보        | 지역별 fraud 분석        |
| zip                   | 우편번호                  | 위치 코드        | 지역 세분화              |
| lat                   | 고객 위도                 | 위치 좌표        | 거리 계산                |
| long                  | 고객 경도                 | 위치 좌표        | 거리 계산                |
| city_pop              | 도시 인구                 | 지역 특성        | 도시 규모 영향 분석      |
| job                   | 직업                      | 사회경제 정보    | 직업별 소비/위험 분석    |
| dob                   | 생년월일                  | 고객 연령 정보   | 나이 기반 분석           |
| trans_num             | 거래 고유 ID              | 식별자           | 로그 추적용 (모델 제외)  |
| unix_time             | Unix 시간                 | 시간 데이터      | 시간 간격 분석           |
| merch_lat             | 가맹점 위도               | 위치 좌표        | 고객-가맹점 거리 계산    |
| merch_long            | 가맹점 경도               | 위치 좌표        | 위치 기반 이상 탐지      |
| is_fraud              | 사기 여부 (0/1)           | 타겟 라벨        | 머신러닝 정답(label)     |
| merch_zipcode         | 가맹점 우편번호           | 위치 코드        | 지역 기반 분석           |



아래에서 **각 필드의 의미 + 분석/모델링 용도**를 실무 관점으로 자세히 알아보겠습니다.

---

# 1. 기본 구조 개요

이 데이터는 크게 5가지 축으로 나뉩니다:

* 📅 시간 정보 (언제)
* 👤 고객 정보 (누가)
* 🏪 거래 정보 (어디서/무엇을)
* 💰 금액 정보 (얼마나)
* 🚨 라벨 정보 (사기 여부)

---

# 2. 컬럼별 상세 설명

---

## 1) `Unnamed: 0`

### 📌 의미

* 보통 CSV 저장 시 자동 생성된 **인덱스 컬럼**
* 실제 금융 의미 없음

### 📌 용도

* ❌ 분석/모델링에는 사용하지 않음
* ✔ 데이터 정렬 확인용

---

## 2) `trans_date_trans_time`

### 📌 의미

* 거래 발생 날짜 + 시간

### 📌 용도 (매우 중요)

시간 기반 패턴 분석 핵심 변수

### 📌 활용

* 시간대별 소비 패턴
* 이상 거래 탐지

### 📌 예시

* 새벽 2시 고액 결제 → fraud 가능성 ↑

---

## 3) `cc_num`

### 📌 의미

* 신용카드 번호 (고객 ID 역할)

### 📌 용도

* 고객 식별자 (Entity Key)

### 📌 분석 활용

* 고객별 소비 패턴 분석
* 고객 기반 aggregation feature 생성

---

## 4) `merchant`

### 📌 의미

* 거래 발생 가맹점 이름

### 📌 용도

* 어떤 상점에서 거래가 발생했는지

### 📌 분석 활용

* 특정 가맹점 사기 비율 분석
* 위험 가맹점 탐지


---

## 5) `category`

### 📌 의미

* 거래 유형 (음식, 쇼핑, 여행 등)

### 📌 용도

* 소비 목적 분류

### 📌 분석 활용

* 소비 패턴 분석
* 이상 거래 탐지

### 📌 예시

* `entertainment` + 고액 결제 → 이상 가능성

---

## 6) `amt`

### 📌 의미

* 거래 금액

### 📌 용도 (핵심 변수)

### 📌 분석 포인트

* 절대값보다 **비율(상대값)**이 중요

---

## 7) `first`, `last`

### 📌 의미

* 고객 이름 (성과 이름)

### 📌 용도

* 고객 식별 보조 정보

### 📌 분석 활용

* 거의 직접 사용하지 않음
* cc_num으로 대체 가능

---

## 8) `gender`

### 📌 의미

* 성별 (M / F)

### 📌 용도

* 고객 그룹 분석

### 📌 활용

* 소비 패턴 차이 분석
* 모델 feature로 사용 가능 (one-hot encoding)

---

## 9) `street`, `city`, `state`, `zip`

### 📌 의미

* 고객 주소 정보

### 📌 용도

* 지역 기반 분석

### 📌 활용

```text
지역별 소비 패턴
도시별 fraud 발생률
```

---

## 10) `lat`, `long`

### 📌 의미

* 고객 거주지 좌표

### 📌 용도

* 위치 기반 분석

### 📌 활용

* 이동 거리 계산
* 비정상 거래 탐지

---

## 11) `city_pop`

### 📌 의미

* 해당 도시 인구 수

### 📌 용도

* 도시 규모 정보

### 📌 활용

* 대도시 vs 소도시 소비 패턴 차이

---

## 12) `job`

### 📌 의미

* 직업 정보

### 📌 용도

* 고객 사회적 특성

### 📌 활용

```text
직업별 평균 소비
직업별 fraud 위험도
```

---

## 13) `dob`

### 📌 의미

* 생년월일

### 📌 용도

* 나이 계산

### 📌 중요성

* 나이별 소비 패턴 분석 핵심

---

## 14) `trans_num`

### 📌 의미

* 거래 고유 ID

### 📌 용도

* 단순 식별자

### 📌 분석

* ❌ 모델 입력 X
* ✔ 로그 추적용

---

## 15) `unix_time`

### 📌 의미

* Unix timestamp (초 단위 시간)

### 📌 용도

* 시간 정렬 및 계산

### 📌 활용

* 거래 간 시간 간격 계산

---

## 16) `merch_lat`, `merch_long`

### 📌 의미

* 가맹점 위치 좌표

### 📌 용도

* 고객 위치 vs 거래 위치 비교


📌 중요 의미:

* 물리적으로 이동 불가능한 거래 탐지

---

## 17) `is_fraud`

### 📌 의미

* 사기 여부 (Target Label)

* 1 = Fraud

* 0 = Normal

### 📌 용도

👉 머신러닝 모델의 정답(label)

---

# 3. 전체 구조 핵심 이해

이 데이터는 본질적으로 다음 구조입니다:

```
고객 (cc_num)
   ↓
시간 (trans_date)
   ↓
가맹점 (merchant)
   ↓
금액 (amt)
   ↓
위치 (lat/long vs merch_lat/long)
   ↓
결과 (is_fraud)
```

# 4. 기본 코드 

```bash
step1.py
# =========================================================
# credit_card_transactions.csv 파일 읽기 및 CSV 로딩
# =========================================================

# 필요한 라이브러리 설치
# !pip install pandas

import pandas as pd
from pathlib import Path

# ---------------------------------------------------------
# 1. CSV 파일 로딩
# ---------------------------------------------------------

csv_path = f"data/credit_card_transactions.csv"

df = pd.read_csv(csv_path)

print("CSV 로딩 완료")

# ---------------------------------------------------------
# 2. 기본 데이터 확인
# ---------------------------------------------------------

print("\n===== 데이터 상위 5개 =====")
print(df.head())

print("\n===== 데이터 정보 =====")
print(df.info())

print("\n===== 결측치 확인 =====")
print(df.isnull().sum())

print("\n===== 기술 통계 =====")
print(df.describe())

# ---------------------------------------------------------
# 3. 날짜 컬럼 변환
# ---------------------------------------------------------

df["trans_date_trans_time"] = pd.to_datetime(
    df["trans_date_trans_time"]
)

print("\n날짜 변환 완료")

# ---------------------------------------------------------
# 4. 분석용 컬럼 추가
# ---------------------------------------------------------

df["year"] = df["trans_date_trans_time"].dt.year
df["month"] = df["trans_date_trans_time"].dt.month
df["day"] = df["trans_date_trans_time"].dt.day
df["hour"] = df["trans_date_trans_time"].dt.hour
df["weekday"] = df["trans_date_trans_time"].dt.day_name()

print("\n분석 컬럼 생성 완료")

# ---------------------------------------------------------
# 5. 기본 거래 분석
# ---------------------------------------------------------

print("\n===== 총 거래 건수 =====")
print(len(df))

print("\n===== 총 거래 금액 =====")
print(df["amt"].sum())

print("\n===== 평균 거래 금액 =====")
print(df["amt"].mean())

print("\n===== 최대 거래 금액 =====")
print(df["amt"].max())

print("\n===== 최소 거래 금액 =====")
print(df["amt"].min())

```

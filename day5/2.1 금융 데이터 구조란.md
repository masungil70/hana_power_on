# 1. 금융 데이터 구조 이해란?

> 금융 데이터 구조 이해 = “돈이 움직이는 데이터를 사람/시간/행위/위험 관점으로 해석하는 능력”

즉, 단순 데이터가 아니라:

* 누가 (Who)
* 언제 (When)
* 어디서 (Where)
* 무엇을 (What)
* 왜 위험한지 (Why)

를 구조적으로 파악하는 것입니다.

---

# 2. 금융 데이터의 기본 구조 5가지 축

금융 데이터는 아래 5가지 구조로 나뉩니다.

1. 고객(CUSTOMER) 구조
2. 거래(TRANSACTION) 구조
3. 시간(TIME) 구조
4. 위치(LOCATION) 구조
5. 위험(RISK) 구조

---

## ① 고객(CUSTOMER) 구조

“누가 거래했는가?”

예:

* cc_num (카드번호)
* gender
* dob(Date of Birth)
* job
* city, state
* city_pop

👉 핵심 의미:

```text id="c9qk3w"
고객 = 행동의 주체 (Entity)
```

---

## ② 거래(TRANSACTION) 구조

“무슨 일이 발생했는가?”

예:

* amt (금액)
* trans_date_trans_time
* merchant(가맹점)
* category

👉 핵심:

```text id="4lq9xg"
거래 = 이벤트(Event)
```

---

## ③ 시간(TIME) 구조

“언제 발생했는가?”

예:

* trans_date_trans_time
* hour
* day
* weekday

👉 금융 데이터 핵심 포인트:

```text id="t7nq9d"
시간은 금융 분석의 가장 중요한 축
```

왜냐하면:

* 새벽 거래 = 이상 가능성 ↑
* 주말 소비 패턴
* 급격한 거래 증가

---

## ④ 위치(LOCATION) 구조

“어디서 발생했는가?”

예:

* lat / long (고객 위치)
* merch_lat / merch_long (가맹점 위치)
* city, state

👉 핵심:

```text id="kq8v4a"
거리 기반 분석 = Fraud 탐지 핵심 요소
```

예:

* 고객 위치 vs 가맹점 거리
* 해외 거래 여부

---

## ⑤ 위험(RISK) 구조

“이 거래가 정상인가?”

예:

* is_fraud (정답 라벨)

👉 머신러닝 핵심:

```text id="9dmp2h"
이 컬럼이 바로 지도학습의 Label
```

---

# 3. 금융 데이터 구조의 특징

1. 시계열 구조
2. 불균형 데이터
3. 관계형 구조
4. 행동 데이터

---

## 특징 1: 시계열 구조

```text id="k1nq2s"
시간 순서가 매우 중요
```

예:

* 하루 10건 → 갑자기 100건
* 새벽 3시 거래

---

## 특징 2: 불균형 데이터

Fraud 데이터:

```text id="xw9l3b"
정상 99%
사기 1%
```

👉 머신러닝에서 가장 중요한 문제

---

## 특징 3: 관계형 구조

하나의 거래는 여러 정보와 연결됨:

```text id="q3v8n1"
고객 ↔ 거래 ↔ 가맹점
```

---

## 특징 4: 행동 데이터

금융 데이터는 단순 숫자가 아니라:

```text id="m2p7xq"
사람의 행동 데이터
```

---


# 4. 실제 예시

---

## 예시 1: 거래 구조 해석

```text id="v9kq2d"
2026-01-01 03:12
고객 A → 편의점 → 500,000원 → is_fraud=1
```

👉 구조:

* 시간: 새벽
* 금액: 비정상적으로 큼
* 위치: 멀리 떨어짐
* 결과: 사기

---

## 예시 2: 정상 거래

```text id="b3xq9m"
14:00 / 점심시간 / 카페 / 5,000원
```

# 5. 금융 데이터 구조 이해 핵심 정리

---

한 문장으로:

```text id="z7nq2p"
금융 데이터 구조 이해 = 사람 + 돈 + 시간 + 위치 + 위험을 하나의 시스템으로 해석하는 능력
```

---

# 6. 금융 데이터 분석 금융 AI, 이상거래 탐지(FDS) 관점에서 구조

금융 데이터 분석에서는 단순히 “거래 데이터가 있다” 수준이 아니라,
데이터를 **어떤 관점으로 구조화**하느냐가 매우 중요합니다.

특히 아래 4가지 구조는 실제 금융 AI, 이상거래 탐지(FDS), 고객 분석, 마케팅, 신용평가에서 핵심이 됩니다.

금융 데이터 구조는:

“고객의 시간 기반 행동(Event)을 수치화(Feature)하고 정답(Label)과 연결하여 AI가 학습할 수 있게 만드는 구조”

- Customer (고객) — “행동의 주체”
- Event 구조(Event Log 구조)
- Feature 구조
- Label 구조

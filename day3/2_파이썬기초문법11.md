# 파이썬 기초문법

## 파이썬 모듈(Module)과 패키지(Package) 기초 문법 정리

파이썬에서 프로그램이 커지면 코드를 파일별로 나누고 재사용해야 합니다.

이때 사용하는 핵심 개념이:

* 모듈(Module)
* 패키지(Package)
* import
* from import

입니다.

---

## 1. 모듈(Module)이란?

### 1-1 개념

파이썬 파일(`.py`) 하나를 모듈이라고 합니다.

예:

```python
math_tools.py
```

이 파일 안에 함수, 변수, 클래스 등을 넣고 다른 파일에서 불러와 사용할 수 있습니다.

---

### 1-2. 왜 모듈을 사용하는가?

### 문제 상황

모든 코드를 한 파일에 작성하면:

* 코드가 길어짐
* 관리 어려움
* 재사용 어려움
* 협업 어려움

---

### 해결 방법

기능별로 파일을 나눔

예:

```text
project/
├── main.py
├── math_tools.py
├── user_service.py
└── db.py
```

## 2. import 기본 문법

### 2-1. 기본 구조

```python
import 모듈이름
```

---

### 2-2. 사용자 모듈 만들기

### 파일명 : math_tools.py

```python

#함수 선언 및 구현 
def add(a, b):
    return a + b

def sub(a, b):
    return a - b
```

---

### 2-3. 다른 파일에서 사용

### main.py

```python

#모듈 함수 호출

result2 = math_tools.sub(10, 5)

#결과 출력 
print(result1)
print(result2)
```

---

```python

5
```

---

```python

```

### 함수 호출 방식

```python

```

형태로 사용합니다.

## 3. from import 문법

### 기본 구조

```python
from 모듈명 import 함수명
```

---

### math_tools.py

```python
def add(a, b):
    return a + b
```

---

### main.py

```python
from math_tools import add

print(add(3, 4))
```

---

### 실행 결과

```python
7
```

---

### 차이점 비교

### import 방식

```python
import math_tools

math_tools.add(1, 2)
```

장점:

* 충돌 방지
* 어떤 모듈 함수인지 명확

단점:

* 코드가 길어짐

---

### from import 방식

```python
from math_tools import add

add(1, 2)
```

장점:

* 코드 짧음
* 편리함

단점:

* 이름 충돌 가능

## 6. 별칭(alias) 사용

### import as

```python
import math_tools as mt

print(mt.add(1, 2))
```

---

### from import as

```python
#math_tools.add() 함수를 plus 변경 
from math_tools import add as plus

print(plus(3, 4))
```

## 9. **name** 과 **main**

매우 중요합니다.

---

### example.py

```python
def hello():
    print("안녕하세요")

# 파이썬의 __name__ 은 현재 실행 중인 모듈(파일)의 이름을 저장하는 특별한 내장 변수 입니다
print(__name__)
```

---

### 직접 실행

```python
python example.py
```

결과:

```python
__main__
```

---

### import로 실행

### main.py

```python
import example
```

결과:

```python
example
```

---

### 활용

```python
if __name__ == "__main__":
    print("직접 실행됨")
```

---

### 의미

모듈을 **직접 실행**하면 **실행** 될 수 있게하고 다른 파일에서 실행하는 경우 실행 안되게 할 수 있습니다.

---

## 실무 필수 패턴

```python
def main():
    print("프로그램 시작")

if __name__ == "__main__":
    main()
```

## 17. 실습 과제

계산기 패키지/모듈/class 만들기

```python
add()
sub()
mul()
div()
```

파이썬에서 계산기 패키지(package)와 모듈(module)을 만드는 가장 기본적인 구조를 예제로 설명하겠습니다.

## 17-1. 프로젝트 패키지(package)와 모듈(module) 구조

### 패키지(package)와 모듈(module) 구조

```text
ex17-1/
├── main.py
└── mycalc/
    ├── __init__.py
    └── calculator.py
```

---

### calculator.py

```python
def add(a, b):
    return a + b

def sub(a, b):
    return a - b

def mul(a, b):
    return a * b

def div(a, b):
    if b == 0:
        raise ValueError("0으로 나눌 수 없습니다.")

    return a / b
```

---

### main.py에서 mycalc 패키지/모듈 사용

```python
from mycalc.calculator import add

print(add(3, 4))
```

---

## 17-2. 프로젝트 객체지향(OOP) 구조

```python
ex17-2/
├── main.py
└── mycalc/
    ├── __init__.py
    └── calculator.py
```

### calculator.py

```python
class Calculator:

    def add(self, a, b):
        return a + b

    def sub(self, a, b):
        return a - b

    def mul(self, a, b):
        return a * b

    def div(self, a, b):
        if b == 0:
            raise ValueError("0으로 나눌 수 없습니다.")

        return a / b
```

---

### main.py에서 mycalc 클래스 사용

```python
from mycalc.calculator import Calculator

calc = Calculator()

print(calc.add(10, 20))
```

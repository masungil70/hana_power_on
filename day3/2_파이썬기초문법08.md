# 파이썬 기초문법

## 파이썬 예외 처리(Exception Handling) 기초 문법 정리

프로그램을 실행하다 보면 다양한 오류(Error)가 발생할 수 있습니다.

예를 들어:

* 숫자를 0으로 나누기
* 존재하지 않는 파일 열기
* 리스트 범위를 벗어난 인덱스 접근
* 숫자가 아닌 문자열을 숫자로 변환

이러한 오류가 발생하면 프로그램이 강제로 종료됩니다.

예외 처리(Exception Handling)는
오류가 발생해도 프로그램이 안전하게 계속 실행되도록 만드는 기술입니다.

---

## 1. 예외 처리 기본 구조

```python
try:
    # 오류가 발생할 가능성이 있는 코드
except:
    # 오류 발생 시 실행되는 코드
```

---

## 2. try

`try` 블록에는 오류가 발생할 가능성이 있는 코드를 작성합니다.

```python
try:
    num = 10 / 0
```

위 코드는 0으로 나누기 때문에 오류 발생:

```python
ZeroDivisionError
```

---

## 3. except

오류가 발생했을 때 실행됩니다.

```python
try:
    num = 10 / 0
except:
    print("오류가 발생했습니다.")
```

실행 결과:

```python
오류가 발생했습니다.
```

프로그램이 종료되지 않고 안전하게 처리됩니다.

---

## 4. finally

오류 발생 여부와 상관없이 항상 실행됩니다.

주로:

* 파일 닫기
* DB 연결 종료
* 네트워크 연결 종료

등에 사용됩니다.

```python
try:
    print("프로그램 실행")
except:
    print("오류 발생")
finally:
    print("프로그램 종료")
```

실행 결과:

```python
프로그램 실행
프로그램 종료
```

---

## 5. 오류 메시지 출력

오류 내용을 변수로 받을 수 있습니다.

```python
try:
    num = 10 / 0
except Exception as e:
    print("오류 내용:", e)
```

실행 결과:

```python
오류 내용: division by zero
```

---

## 6. 자주 사용하는 예외 종류

| 예외              | 설명               |
| ----------------- | ----------         |
| ZeroDivisionError | 0으로 나눔         |
| ValueError        | 잘못된 값          |
| TypeError         | 자료형 오류        |
| IndexError        | 리스트 인덱스 오류 |
| KeyError          | 딕셔너리 키 오류   |
| FileNotFoundError | 파일 없음          |

---

## 7. 예제

### 7-1. 0으로 나누기

```python
try:
    num = 10 / 0
except:
    print("0으로 나눌 수 없습니다.")
```

실행 결과:

```python
0으로 나눌 수 없습니다.
```

---

### 7-2. 문자열 -> 숫자 변환 오류

```python
try:
    num = int("hello")
except:
    print("숫자로 변환할 수 없습니다.")
```

실행 결과:

```python
숫자로 변환할 수 없습니다.
```

---

### 7-3. 리스트 인덱스 오류

```python
numbers = [1, 2, 3]

try:
    print(numbers[5])
except:
    print("인덱스 범위를 벗어났습니다.")
```

실행 결과:

```python
인덱스 범위를 벗어났습니다.
```

---

### 7-4. finally 사용

```python
try:
    print("파일 열기")
except:
    print("오류 발생")
finally:
    print("파일 닫기")
```

실행 결과:

```python
파일 열기
파일 닫기
```

---

### 7-5. 특정 예외만 처리

```python
try:
    num = int(input("숫자 입력: "))
    result = 100 / num
    print(result)

except ZeroDivisionError:
    print("0으로 나눌 수 없습니다.")

except ValueError:
    print("숫자만 입력하세요.")
```

설명:

* 0 입력 → ZeroDivisionError
* 문자 입력 → ValueError

---

### 7-6. 오류 메시지 출력

```python
try:
    numbers = [1, 2, 3]
    print(numbers[10])

except Exception as e:
    print("오류 발생:", e)
```

실행 결과:

```python
오류 발생: list index out of range
```

---

### 7-7. 정상 실행 시 else 사용

```python
try:
    num = int(input("숫자 입력: "))
except ValueError:
    print("숫자 입력 오류")
else:
    print("입력한 숫자:", num)
finally:
    print("프로그램 종료")
```

설명:

* 오류 없으면 `else` 실행
* 마지막에 `finally` 항상 실행

---

### 7-8. 파일 열기 예외 처리

```python
try:
    file = open("data.txt", "r")
    content = file.read()
    print(content)

except FileNotFoundError:
    print("파일이 존재하지 않습니다.")

finally:
    print("작업 종료")
```

---

### 7-9. 여러 예외 동시에 처리

```python
try:
    num = int(input("숫자 입력: "))
    result = 10 / num

except (ValueError, ZeroDivisionError) as e:
    print("오류 발생:", e)
```

설명:

* 여러 예외를 한 번에 처리 가능

---

### 7-10. 사용자 정의 예외 발생

`raise`를 사용하면 직접 오류를 발생시킬 수 있습니다.

```python
age = -1

try:
    if age < 0:
        raise ValueError("나이는 음수가 될 수 없습니다.")

except ValueError as e:
    print(e)
```

실행 결과:

```python
나이는 음수가 될 수 없습니다.
```

---

### 7-11. 강제 예외 발생 입력 검증

```python
while True:
    try:
        num = int(input("1~10 숫자 입력: "))

        if num < 1 or num > 10:
            raise ValueError("범위를 벗어났습니다.")

        print("입력 성공:", num)
        break

    except ValueError as e:
        print("오류:", e)
```

설명:

* 숫자 검증
* 범위 검증
* 오류 메시지 출력
* 정상 입력 시 종료

---

### 7-12. API 호출시 예외 처리 방법

```python
try:
    # 서버 요청
    print("API 호출")

except TimeoutError:
    print("서버 응답 시간 초과")

except ConnectionError:
    print("서버 연결 실패")
```

---

### 7-13. 데이터 변환

```python
data = ["10", "20", "hello", "30"]

for item in data:
    try:
        print(int(item))
    except ValueError:
        print("변환 실패:", item)
```

---

## 8. 예외 처리 핵심 정리

| 문법           | 설명                |
| -------------- | -----------         |
| try            | 오류 발생 가능 코드 |
| except         | 오류 처리           |
| else           | 오류 없을 때 실행   |
| finally        | 항상 실행           |
| raise          | 강제로 예외 발생    |
| Exception as e | 오류 메시지 저장    |

---

## 9. 좋은 예외 처리 방법

```python
except ValueError:
```

처럼 구체적으로 작성하는 것이 좋습니다.

---

## 10. 좋지 않은 방법

```python
except:
```

모든 오류를 무조건 처리하면
문제 원인을 찾기 어려워집니다.

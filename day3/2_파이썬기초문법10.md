# 파이썬 기초문법

## 파이썬 객체지향(OOP) 기초 문법 정리

객체지향(Object Oriented Programming, OOP)은
**데이터와 기능을 하나로 묶어서 관리하는 프로그래밍 방식**입니다.

파이썬에서는 OOP를 매우 많이 사용합니다.

예:

* 웹 개발
* 게임 개발
* AI 모델 관리
* FastAPI / Django
* Kubernetes 관리 도구
* 데이터 처리 시스템

등 대부분의 실무 프로젝트에서 사용됩니다.

---

## 1. 객체지향(OOP)이란?

예를 들어 자동차를 생각해봅시다.

자동차는:

* 색상
* 속도
* 모델명

같은 데이터(속성)가 있고,

* 출발하기
* 정지하기
* 가속하기

같은 기능(함수, 메소드)이 있습니다.

OOP에서는 이것을 하나의 객체(Object)로 표현합니다.

---

## 2. 클래스(Class)

클래스는 객체를 만들기 위한 설계도입니다.
사용자 정의 자료형 이라고 생각해도 됩니다.

예:

* 붕어빵 틀 = 클래스
* 실제 붕어빵 = 객체

---

### 2-1. 클래스 기본 문법

```python
class 클래스이름:
    내용
```

## 3. 객체(Object)

클래스를 이용해서 실제 데이터를 생성한 것을 객체라고 합니다.
클래스를 이용하여 선언된 변수라고 생각해도 됩니다

---

### 3-1. 가장 기본 클래스 만들기

```python
class Dog:
    pass
```

설명:

* `Dog`라는 클래스를 생성
* `pass`는 비어있다는 의미

---


## 5. self란?

`self`는 메소드 내부에서 자기 자신 객체를 의미합니다.

매우 중요합니다.

```python
class Dog:

    def hello(self):
        print("나는 강아지")

dog1 = Dog()
dog1.hello()
```

실제로는 내부적으로 이렇게 동작합니다.

```python
Dog.hello(dog1)
```

즉:

* self = 현재 객체

### 9-1. 학생 관리 클래스

```python
#클래스 선언 
class Student:
    #생성자 정의 
    def __init__(self, name, age):
        self.name = name
```

### 9-2. 계산기 클래스

### 9-3. 은행 계좌 클래스

```python
#클래스 선언 
class BankAccount:

    #생성자 정의 
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

    #입금 메서드 정의 
    def deposit(self, money):
        self.balance += money
        print(f"{money}원 입금 완료")

    #출금 메서드 정의
    def withdraw(self, money):

        if self.balance >= money:
            self.balance -= money
            print(f"{money}원 출금 완료")
        else:
            print("잔액 부족")

    #잔액 확인 메서드 정의 
    def show_balance(self):
        print(f"현재 잔액: {self.balance}원")

#객체 생성
account = BankAccount("김철수", 10000)

#객체를 이용한 메서드 호출
account.deposit(5000)

#출금 메서드 호출
account.withdraw(3000)

#잔액 확인 메서드 호출
account.show_balance()
```

결과:

```python
5000원 입금 완료
3000원 출금 완료
현재 잔액: 12000원
```

## 9-6. 캡슐화(Encapsulation, 은닉화) 개념

데이터를 보호하는 방식입니다.

목적 :

* 외부 직접 접근을 "막는 것"이 아니라
* "실수로 접근하지 못하게 하는 것"

에 가깝습니다.

---

### 접근 수준 정리

| 표기     | 의미               | 접근 수준      |
| -------- | ----------------   | ------------   |
| `name`   | 일반 변수          | public         |
| `_name`  | 내부용 관례        | protected 비슷 |
| `__name` | Name Mangling 적용 | private 비슷   |

```python
#클래스 선언
class User:

    #생성자 정의 
    def __init__(self, name):
        self.__name = name

    #getter 메서드 정의 
    def get_name(self):
        return self.__name

user = User("홍길동")

print(user.get_name())
```

결과:

```python
홍길동
```

## 13. super()

부모 클래스의 메소드/속성을 호출

```python
class Animal:

    def __init__(self, name):
        self.name = name

class Dog(Animal):

    def __init__(self, name, age):

        super().__init__(name)

        self.age = age

dog = Dog("초코", 3)

print(dog.name)
print(dog.age)
```

결과:

```python
초코
3
```

## 14. 예제 코드

### 14-1. 회원 관리 시스템

```python
#클래스 선언 
class User:

    #생성자 정의 
    def __init__(self, username, password):
        self.username = username
        self.password = password

    #로그인 메서드 정의
    def login(self):
        print(f"{self.username} 로그인 완료")

    #로그아웃 메서드 정의
    def logout(self):
        print(f"{self.username} 로그아웃 완료")

#객체 생성
user1 = User("admin", "1234")

#객체를 이용한 메서드 호출
user1.login()
user1.logout()
```
